QUICK START COMMANDS:
================================================================================

# 1. Go to your script directory
cd /home/tmfestival/backups/script

# 2. Clean up old files (optional, setup script does this)
bash setup-github.sh

# 3. Initialize Git repository
git init

# 4. Add all files
git add .

# 5. Create first commit
git commit -m "Initial commit: WordPress Backup, Migration & Restore Tool v1.0.0"

# 6. Create repository on GitHub
# Go to: https://github.com/new
# Repository name: wp-backup
# Description: Complete WordPress backup, migration & restore tool. Single standalone script with WP-CLI integration.
# Visibility: Public
# Do NOT initialize with README, .gitignore, or license (we already have these)
# Click "Create repository"

# 7. Connect to GitHub
git remote add origin https://github.com/gabirosca/wp-backup.git
git branch -M main

# 8. Push to GitHub
git push -u origin main

# You'll be prompted for credentials:
# Username: gabirosca
# Password: [Use a GitHub Personal Access Token]

# Get token at: https://github.com/settings/tokens
# Create new token with 'repo' scope

================================================================================
TROUBLESHOOTING:
================================================================================

ERROR: Permission denied
SOLUTION: Use Personal Access Token instead of password
          Get token: https://github.com/settings/tokens

ERROR: Repository already exists
SOLUTION: Delete the repository on GitHub and start over
          OR use: git remote set-url origin https://github.com/gabirosca/wp-backup.git

ERROR: Files not showing on GitHub
SOLUTION: Check .gitignore isn't excluding them
          Run: git status
          Ensure files are staged: git add [filename]
